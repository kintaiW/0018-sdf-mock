# SDF Mock 服务错误码文档

> 标准依据：GM/T 0018—2012 《密码设备应用接口规范》

---

## 1. 返回码格式说明

SDF 接口的所有函数均返回一个 `int` 类型的返回值：

| 返回值 | 含义 |
|--------|------|
| `0` （`SDR_OK`）| 操作成功 |
| 非零值 | 操作失败，具体错误见下表 |

错误码统一分布在 `0x01000001` ~ `0x01000013` 区间（GM/T 0018 定义），Mock 服务完整实现上述全部错误码。

## 2. 错误码速查表

### 2.1 成功码

| 错误码（十六进制） | 名称 | 含义 |
|-------------------|------|------|
| `0x00000000` | `SDR_OK` | 操作成功，无错误 |

### 2.2 通用与设备错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x01000001` | `SDR_UNKNOWERR` | 未知错误 | 发生了未归类的内部异常 | 开启 `RUST_LOG=debug` 查看详细日志；检查调用参数合法性 |
| `0x01000002` | `SDR_NOTSUPPORT` | 功能不支持 | 调用的功能或参数组合在当前 Mock 版本中未实现 | 查阅 README.md 中的支持功能列表；确认是否使用了实验性接口 |
| `0x01000003` | `SDR_COMMFAIL` | 通信失败 | 与设备（Mock 服务）的通信中断；网络异常 | 确认 Mock 服务正在运行；检查监听地址和端口；查看服务端日志 |
| `0x01000004` | `SDR_HARDFAIL` | 硬件失败 | 设备硬件故障（真实设备场景）；Mock 服务内部状态异常 | 重启 Mock 服务；若问题持续，查看 panic 日志并提交 Issue |
| `0x01000005` | `SDR_OPENDEVICEERR` | 打开设备失败 | Mock 服务未启动；设备已被其他进程占用；资源耗尽 | 确认服务已启动；检查是否有多个进程同时连接且超出最大会话数 |
| `0x01000006` | `SDR_OPENSESSIONERR` | 打开会话失败 | 设备未正确打开；超过最大并发会话数限制 | 确认已成功调用 `SDF_OpenDevice`；检查并发会话数是否超限 |

### 2.3 参数与数据错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x01000007` | `SDR_PARDATAERR` | 参数数据错误 | 传入参数值非法（如密钥长度不合规、NULL 指针）；参数类型不匹配 | 逐一检查 API 参数；对照 GM/T 0018 规范确认参数取值范围 |
| `0x01000008` | `SDR_INDATALENERR` | 输入数据长度错误 | 输入数据长度为零、负数或超出接口限制 | SM4 分组密钥长度必须为 128 位（16字节）；SM2 输入明文有长度上限 |
| `0x01000009` | `SDR_INDATAERR` | 输入数据错误 | 数据格式不合法（如 Base64 非法字符）；密文损坏；结构体字段填写错误 | 检查数据编码格式；确认密文来源与加密时使用的算法一致 |

### 2.4 随机数错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x0100000A` | `SDR_GENRANDERR` | 生成随机数失败 | 系统熵源不足；Mock 服务内部随机数生成器异常 | Mock 环境下通常不会触发；若出现，重启服务后重试 |

### 2.5 哈希运算错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x0100000B` | `SDR_HASHNOTSUPPORT` | 哈希算法不支持 | 传入的哈希算法标识不在支持列表中（Mock 支持 SM3） | 确认算法标识符为 `SGD_SM3`（值 `0x00000001`） |
| `0x0100000C` | `SDR_HASHERR` | 哈希运算失败 | 哈希会话状态异常；数据输入顺序错误 | 确认调用顺序：`SDF_HashInit` → `SDF_HashUpdate` → `SDF_HashFinal` |

### 2.6 签名与验签错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x0100000D` | `SDR_SIGNERR` | 签名失败 | 私钥不存在或已损坏；输入数据格式错误 | 确认密钥已通过 `SDF_GenerateKeyPair_ECC` 生成；检查私钥索引号是否正确 |
| `0x0100000E` | `SDR_VERIFYERR` | 验签失败 | 签名值与数据/公钥不匹配；数据在传输中被篡改 | 确认签名和原文使用了相同的公钥；检查数据是否在传递过程中发生了编码变换 |

### 2.7 对称加解密错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x0100000F` | `SDR_SYMENCERR` | 对称加密失败 | 密钥句柄无效；数据长度不满足分组要求 | 确认对称密钥已通过 `SDF_GenerateKey` 或导入方式获取；SM4 需 16 字节对齐 |
| `0x01000010` | `SDR_SYMDECERR` | 对称解密失败 | 密文长度错误；密钥与加密时不匹配；填充数据损坏 | 确认解密密钥与加密密钥一致；检查密文是否完整传递 |

### 2.8 密钥管理错误

| 错误码（十六进制） | 名称 | 含义 | 可能原因 | 解决方案 |
|-------------------|------|------|----------|----------|
| `0x01000011` | `SDR_KEYNOTEXIST` | 密钥不存在 | 指定索引的密钥从未生成；密钥已被删除；索引号错误 | 先调用 `SDF_GenerateKeyPair_ECC` 生成密钥；确认使用正确的密钥索引 |
| `0x01000013` | `SDR_ALGNOTSUPPORT` | 算法不支持 | 传入的算法标识符不在 Mock 服务支持范围内 | Mock 服务支持算法：SM2（非对称）、SM3（哈希）、SM4（对称）；确认 `AlgID` 参数值 |

## 3. 标准算法标识符参考

| 算法标识符 | 值（十六进制） | 说明 |
|-----------|---------------|------|
| `SGD_SM1_ECB` | `0x00000101` | SM1 ECB 模式 |
| `SGD_SM1_CBC` | `0x00000102` | SM1 CBC 模式 |
| `SGD_SM4_ECB` | `0x00000401` | SM4 ECB 模式 |
| `SGD_SM4_CBC` | `0x00000402` | SM4 CBC 模式 |
| `SGD_SM2` | `0x00020200` | SM2 椭圆曲线 |
| `SGD_SM3` | `0x00000001` | SM3 哈希 |

> Mock 服务当前主要支持 SM2、SM3、SM4，若使用 SM1 相关标识符可能返回 `SDR_ALGNOTSUPPORT`。

## 4. 调试技巧

### 4.1 开启详细日志

```bash
# 开发调试：输出所有级别日志
RUST_LOG=debug cargo run

# 仅显示关键错误
RUST_LOG=error cargo run

# 按模块细粒度控制（推荐）
RUST_LOG=sdf_mock=debug,warn cargo run
```

日志输出格式示例：

```
[DEBUG sdf_mock::session] SDF_OpenSession: device_handle=1, session_id=42
[DEBUG sdf_mock::crypto]  SDF_InternalSign_ECC: key_index=1, data_len=32
[ERROR sdf_mock::crypto]  SDR_KEYNOTEXIST: key index 99 not found
```

### 4.2 会话生命周期调试

SDF 接口有严格的调用层级，常见错误由调用顺序错误导致：

```
SDF_OpenDevice(phDeviceHandle)          ← 必须第一步
      │
      └─ SDF_OpenSession(hDeviceHandle) ← 在设备句柄上开会话
            │
            ├─ SDF_GenerateRandom       ← 业务操作
            ├─ SDF_HashInit/Update/Final
            ├─ SDF_InternalSign_ECC
            └─ SDF_CloseSession         ← 操作完成后关闭
      │
      └─ SDF_CloseDevice               ← 最后关闭设备
```

### 4.3 使用集成测试验证

```bash
# 运行全部集成测试
cargo test --test '*'

# 运行指定测试（如签名相关）
cargo test sign -- --nocapture
```

### 4.4 错误码十六进制转换

```bash
# Shell 中快速查看错误码十进制/十六进制对应
printf "0x%08X\n" 16777217   # 输出：0x01000001
printf "%d\n" 0x0100000E     # 输出：16777230
```

## 5. 常见问题

### Q1：调用 `SDF_OpenDevice` 返回 `SDR_OPENDEVICEERR (0x01000005)`

**排查步骤：**
```
1. 确认 SDF Mock 服务已启动
   cargo run --bin sdf-mock

2. 确认客户端连接参数（IP/端口/协议）与服务一致

3. 检查是否有防火墙规则阻止连接

4. 查看 Mock 服务控制台输出是否有错误信息
```

### Q2：哈希计算返回 `SDR_HASHERR (0x0100000C)`

**正确调用顺序：**
```c
// 正确示例
SDF_HashInit(hSession, SGD_SM3, NULL, NULL, 0);
SDF_HashUpdate(hSession, pData, dataLen);
SDF_HashFinal(hSession, pHash, &hashLen);

// 错误示例（跳过 Init 直接 Update）
SDF_HashUpdate(hSession, pData, dataLen);  // → SDR_HASHERR
```

### Q3：SM4 解密返回 `SDR_SYMDECERR (0x01000010)`

**检查清单：**
- [ ] 解密密钥是否与加密时相同？
- [ ] CBC 模式下 IV 是否与加密时相同？
- [ ] 密文长度是否为 16 的整数倍？
- [ ] 密文数据在传输中是否发生了字节丢失或字符集转换？

### Q4：SM2 验签返回 `SDR_VERIFYERR (0x0100000E)`

**常见根本原因：**
- 签名时对数据做了 SM3 哈希，但验签时传入了原始数据（或反之）
- 公钥坐标 X/Y 字节序（大端/小端）与签名方不一致
- 数据经过了 Base64 编码/解码，导致内容不一致

### Q5：同时运行多个测试进程时出现 `SDR_OPENSESSIONERR`

Mock 服务默认支持有限数量的并发会话，并发测试时应注意：
```bash
# 串行运行测试避免并发冲突
cargo test -- --test-threads=1
```

## 6. 与其他 Mock 服务联动

SDF Mock 通常作为密码运算的核心服务，与 SKF Mock（GM/T 0016）和 SVS Mock（GM/T 0029）协同工作：

```
客户端应用
    │
    ├─ SKF Mock (0016)  → 智能密码钥匙操作（PIN 验证、密钥管理）
    │
    ├─ SDF Mock (0018)  → 密码运算（签名、加解密、哈希）← 当前文档
    │
    └─ SVS Mock (0029)  → 签名验签服务（证书链验证、OCSP）
```

当 SVS 返回签名验签失败时，可能需要同时查看 SDF 层的 `SDR_VERIFYERR` 日志来定位根本原因。
